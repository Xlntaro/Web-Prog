# Business Requirements Document (BRD)
## Project: Скарбничка (Expense Tracker)

### 1. Project Overview
Проєкт "Скарбничка" — це веб-додаток для керування особистими фінансами. Він дозволяє користувачам швидко додавати свої щоденні витрати, категоризувати їх та відслідковувати загальний баланс у зручному інтерфейсі.

### 2. Target Audience
- Студенти, які хочуть контролювати свої кишенькові витрати.
- Фрілансери та самозайняті особи для базового фінансового обліку.
- Будь-який користувач, який шукає простий інструмент без складної реєстрації.

### 3. Business Objectives
- **Зручність:** Надати максимально швидкий спосіб фіксування витрат (не більше 3 кліків).
- **Візуалізація:** Забезпечити користувача чітким розумінням загального фінансового стану.
- **Доступність:** Додаток має працювати прямо в браузері без необхідності встановлення.

### 4. Functional Requirements
- **FR1 (Додавання витрат):** Користувач повинен мати можливість додати нову транзакцію, вказавши назву, суму та категорію.
- **FR2 (Перегляд історії):** Система повинна відображати список усіх доданих транзакцій у зворотному хронологічному порядку.
- **FR3 (Видалення):** Користувач може видалити помилково додану транзакцію.
- **FR4 (Підрахунок балансу):** Система повинна автоматично розраховувати та відображати загальну суму всіх витрат.
- **FR5 (Очищення даних):** Можливість повністю стерти всі дані через меню налаштувань.

### 5. Non-Functional Requirements
- **NFR1 (Performance):** Додаток має завантажуватися миттєво (Single Page Application).
- **NFR2 (Usability):** Інтерфейс повинен бути інтуїтивно зрозумілим і використовувати сучасний дизайн (Glassmorphism).
- **NFR3 (Persistence):** Дані повинні зберігатися локально на пристрої користувача між сесіями.
