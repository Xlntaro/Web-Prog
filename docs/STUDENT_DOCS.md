# ДОКУМЕНТАЦІЯ ПРОЄКТУ (ПОЯСНЮВАЛЬНА ЗАПИСКА)

## Вступ

**Мета роботи:** розробка сучасного та швидкого веб-додатка "Скарбничка" для ефективного обліку та контролю особистих фінансів користувача.

**Завдання:**
1. Проаналізувати предметну область та визначити ключові вимоги до системи обліку витрат.
2. Обрати оптимальний технологічний стек для реалізації клієнтської частини додатку.
3. Спроєктувати зручний користувацький інтерфейс (UI) з використанням сучасних тенденцій веб-дизайну.
4. Розробити логіку роботи Single Page Application (SPA) з підтримкою маршрутизації.
5. Реалізувати механізм локального збереження даних користувача.

**Об'єкт дослідження:** процес управління та обліку особистих фінансових ресурсів.

**Предмет дослідження:** програмні засоби та веб-технології для створення інтерактивних клієнтських додатків (SPA) у сфері фінансового обліку.

**Актуальність теми:** У сучасному світі фінансова грамотність та контроль над власними витратами є надзвичайно важливими навичками. Попри велику кількість існуючих додатків, багато з них є перевантаженими функціоналом, вимагають платної підписки або містять нав'язливу рекламу. Тому створення легкого, безкоштовного та швидкого веб-додатка, який працює безпосередньо в браузері та не вимагає встановлення, є дуже актуальним завданням, особливо для студентів та молоді.

**Практичне значення:** Розроблений веб-додаток "Скарбничка" є повністю готовим до використання інструментом. Його можна застосовувати у повсякденному житті для фіксації щоденних витрат, аналізу балансу та формування звички контролювати власні фінанси.

## Умовні позначення

- **API** (Application Programming Interface) — набір визначень та протоколів для взаємодії між різними програмними компонентами.
- **CSS** (Cascading Style Sheets) — каскадні таблиці стилів, спеціальна мова, що використовується для опису зовнішнього вигляду сторінок, написаних мовами розмітки даних.
- **DOM** (Document Object Model) — об'єктна модель документа, програмний інтерфейс для HTML та XML документів.
- **HTML** (HyperText Markup Language) — стандартизована мова розмітки гіпертексту документів для Всесвітньої павутини.
- **JS** (JavaScript) — динамічна, об'єктно-орієнтована мова програмування, що використовується для створення інтерактивних веб-сторінок.
- **SPA** (Single Page Application) — односторінковий веб-додаток, який завантажує єдиний HTML-документ і динамічно оновлює його під час взаємодії з користувачем.
- **UI** (User Interface) — користувацький інтерфейс, сукупність засобів і методів, за допомогою яких користувач взаємодіє з програмою.

## Розділ 1. Опис технологічного стеку

Для реалізації веб-додатка "Скарбничка" було обрано сучасний стек нативних веб-технологій без використання важких зовнішніх фреймворків чи бібліотек (таких як React, Angular або Vue.js). Такий підхід дозволив забезпечити максимальну швидкодію, легкість підтримки коду та продемонструвати глибоке розуміння фундаментальних принципів веб-розробки.

**HTML5 (HyperText Markup Language 5)**
Як основа для структуризації контенту використано стандарт HTML5. Він забезпечує семантичну розмітку документа, що позитивно впливає на доступність (accessibility) додатку та його коректне відображення в усіх сучасних браузерах. У проєкті активно використовуються такі семантичні теги, як `<header>`, `<main>`, `<nav>`, `<aside>` та `<section>`, що робить структуру коду логічною та зрозумілою.

**CSS3 (Cascading Style Sheets 3)**
Для оформлення зовнішнього вигляду додатку застосовано CSS3. Особливістю візуального стилю є використання сучасного напрямку веб-дизайну — **Glassmorphism** (ефект напівпрозорого матового скла). Цей ефект досягається за допомогою властивостей `backdrop-filter: blur()`, напівпрозорих фонових кольорів (`rgba`) та делікатних тіней (`box-shadow`).
Крім того, широко використані:
- **CSS Variables (Змінні):** для централізованого управління кольоровою палітрою та розмірами, що значно спрощує підтримку та зміну теми (наприклад, реалізацію Dark Mode).
- **Flexbox та CSS Grid:** для створення гнучких, адаптивних макетів (layout), які коректно відображаються на пристроях з різною роздільною здатністю екрану.
- **CSS Animations:** для створення плавних мікро-взаємодій (hover ефекти, плавна поява елементів списку, анімований фон), що робить користувацький досвід більш приємним.

**Vanilla JavaScript (ECMAScript 6+)**
Вся логіка додатку написана на чистому JavaScript з використанням сучасних стандартів ES6+. Ключовою архітектурною особливістю є використання **Native ES Modules** (`import` / `export`), що дозволило розбити код на логічні модулі (компоненти, сторінки, роутер, стор) без використання інструментів збірки (наприклад, Webpack або Vite). 
У проєкті реалізовано:
- **Кастомний SPA Router:** перехоплює події навігації та використовує History API (`history.pushState`) для динамічного підвантаження потрібного екрану без перезавантаження сторінки.
- **State Management (Керування станом):** створено простий реактивний об'єкт `Store` (на базі патерну Observer), який зберігає всі транзакції та сповіщає компоненти про необхідність оновлення інтерфейсу при зміні даних.

**Web Storage API (LocalStorage)**
Оскільки додаток є повністю клієнтським і не взаємодіє з сервером бази даних, для постійного збереження інформації (persistence) використовується `localStorage`. Це дозволяє зберігати історію витрат у форматі JSON безпосередньо в браузері користувача. Дані не втрачаються після закриття вкладки або оновлення сторінки, що забезпечує безперервність роботи з додатком.

## Висновки

У результаті виконання роботи було успішно спроєктовано та розроблено веб-додаток "Скарбничка" для обліку особистих фінансів. Всі поставлені завдання були виконані в повному обсязі. 

Створений додаток має сучасний користувацький інтерфейс, побудований за принципами Glassmorphism, що забезпечує високу зручність використання. Завдяки архітектурі Single Page Application (SPA) та використанню нативних ES-модулів, додаток працює надзвичайно швидко і не потребує завантаження додаткових ресурсів під час переходів між сторінками. Використання LocalStorage гарантує надійне збереження даних користувача на клієнтській стороні. 

Таким чином, розроблений програмний продукт повністю відповідає заявленій меті та може бути використаний на практиці для ефективного ведення особистого бюджету.
