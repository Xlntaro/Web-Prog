# ДОКУМЕНТАЦІЯ ПРОЄКТУ (ПОЯСНЮВАЛЬНА ЗАПИСКА)

## Вступ

**Мета роботи:** проєктування та розробка сучасного, високопродуктивного та інтерактивного веб-додатка "Скарбничка" (Expense Tracker), призначеного для ефективного обліку, моніторингу та контролю особистих фінансів користувача без необхідності використання складних бухгалтерських систем.

**Завдання:**
1. Провести аналіз предметної області та визначити ключові функціональні та нефункціональні вимоги до системи персонального фінансового обліку.
2. Дослідити існуючі програмні рішення та обґрунтувати вибір оптимального технологічного стеку (HTML5, CSS3, Vanilla JS) для реалізації клієнтської частини додатку.
3. Спроєктувати інтуїтивно зрозумілий та ергономічний користувацький інтерфейс (UI) з використанням сучасних візуальних тенденцій, зокрема стилю Glassmorphism.
4. Розробити архітектуру Single Page Application (SPA) для забезпечення плавної навігації без перезавантаження сторінок браузера.
5. Реалізувати алгоритми обробки фінансових даних, включаючи додавання, видалення та агрегацію транзакцій у реальному часі.
6. Впровадити надійний механізм локального збереження даних (persistence) із застосуванням Web Storage API, щоб забезпечити автономність роботи додатку.

**Об'єкт дослідження:** процес управління, розподілу та обліку особистих фінансових ресурсів в умовах повсякденної життєдіяльності.

**Предмет дослідження:** сучасні програмні засоби, архітектурні патерни та веб-технології для створення інтерактивних клієнтських додатків (SPA) у сфері фінансового моніторингу.

**Актуальність теми:** У сучасному світі фінансова грамотність та контроль над власними витратами є надзвичайно важливими навичками для забезпечення стабільності та досягнення фінансових цілей. Попри велику кількість існуючих мобільних та веб-додатків для обліку фінансів, багато з них є перевантаженими надлишковим функціоналом, вимагають обов'язкової реєстрації, платної підписки або містять нав'язливу рекламу. Збір та обробка персональних фінансових даних на сторонніх серверах також викликає занепокоєння щодо конфіденційності. Тому створення легкого, безкоштовного та швидкого веб-додатка, який працює безпосередньо в браузері, зберігає дані виключно на пристрої користувача та не вимагає встановлення, є надзвичайно актуальним завданням, особливо для студентів, фрілансерів та молоді.

**Практичне значення:** Розроблений веб-додаток "Скарбничка" є повністю готовим до використання інструментом. Його можна застосовувати у повсякденному житті для швидкої фіксації щоденних витрат, аналізу поточного балансу та формування корисної звички контролювати власні фінанси. Відкритість архітектури додатку дозволяє легко масштабувати його в майбутньому, додаючи нові модулі (наприклад, графіки, експорт даних чи інтеграцію з хмарними сервісами).

## Умовні позначення

- **API** (Application Programming Interface) — прикладний програмний інтерфейс, набір визначень та протоколів для взаємодії між компонентами системи.
- **CSS** (Cascading Style Sheets) — каскадні таблиці стилів, мова для опису зовнішнього вигляду документа.
- **DOM** (Document Object Model) — об'єктна модель документа, програмний інтерфейс для доступу та зміни вмісту HTML-сторінок.
- **ES6+** (ECMAScript 2015 і новіші) — сучасні стандарти мови програмування JavaScript.
- **HTML** (HyperText Markup Language) — стандартизована мова розмітки гіпертексту документів для веб-середовища.
- **JS** (JavaScript) — високорівнева динамічна мова програмування, що забезпечує інтерактивність веб-додатків.
- **SPA** (Single Page Application) — односторінковий веб-додаток, який виконується повністю на стороні клієнта і не вимагає повного перезавантаження сторінки при переході між розділами.
- **UI / UX** (User Interface / User Experience) — користувацький інтерфейс та досвід користувача.

## Розділ 1. Опис технологічного стеку

Для реалізації веб-додатка "Скарбничка" було прийнято рішення відмовитися від використання важких зовнішніх фреймворків (таких як React, Angular або Vue.js) та інструментів збірки (Webpack, Rollup). Натомість було обрано сучасний стек "чистих" (vanilla) нативних веб-технологій. Такий підхід має ряд суттєвих переваг: він дозволяє забезпечити максимальну швидкодію додатку завдяки відсутності накладних витрат фреймворків, робить код більш прозорим для розуміння та демонструє глибоке володіння фундаментальними принципами веб-розробки.

### 1.1. Архітектурний підхід Single Page Application (SPA)
Додаток спроєктовано за моделлю SPA. На відміну від традиційних багатосторінкових додатків (Multi-Page Applications), де кожен перехід за посиланням супроводжується запитом до сервера і повним перемальовуванням HTML-сторінки, SPA завантажує єдиний початковий HTML-документ. Усі подальші зміни інтерфейсу (переходи між екранами "Головна", "Транзакції", "Налаштування") відбуваються динамічно на стороні клієнта за допомогою JavaScript. Це створює ілюзію роботи десктопної або мобільної програми, забезпечуючи миттєвий відгук на дії користувача та значно покращуючи загальний UX.

### 1.2. Розмітка та семантика: HTML5
Як основа для структуризації контенту використано останній стандарт мови розмітки — HTML5. Основний акцент було зроблено на семантичності коду. Замість повсюдного використання загальних контейнерів `<div>`, у проєкті застосовуються теги, що чітко описують призначення свого вмісту: `<header>`, `<main>`, `<nav>`, `<aside>`, `<section>` та `<footer>`. Це має подвійний ефект: 
По-перше, семантична розмітка критично важлива для веб-доступності (accessibility, a11y), оскільки програми читання з екрану (screen readers) можуть коректно інтерпретувати структуру сторінки для людей з вадами зору.
По-друге, це покращує читабельність коду для розробників, роблячи DOM-дерево логічним і передбачуваним.

### 1.3. Візуальне оформлення: CSS3 та Glassmorphism
Для створення сучасного та преміального зовнішнього вигляду додатку застосовано можливості стандарту CSS3. Особливістю візуального стилю є використання напрямку **Glassmorphism** (ефект напівпрозорого матового скла). 
Цей ефект реалізується за допомогою CSS-властивості `backdrop-filter: blur(16px)`, яка розмиває елементи, що знаходяться позаду поточного контейнера. У поєднанні з напівпрозорим фоном (`background: rgba(255, 255, 255, 0.05)`) та делікатними світлими рамками це створює відчуття глибини, об'єму та "повітряності" інтерфейсу.
У таблицях стилів широко використані:
- **CSS Custom Properties (Змінні):** `var(--primary-color)` дозволяють централізовано керувати кольоровою палітрою, що значно спрощує підтримку та відкриває шлях до легкого впровадження тематизації (наприклад, світлої/темної теми).
- **CSS Grid Layout та Flexbox:** дві найпотужніші системи компонування, які використані для побудови адаптивних макетів (responsive design). Завдяки їм бокова панель (sidebar) та контентна частина коректно поводяться при зміні розміру вікна.
- **CSS Animations & Transitions:** для створення плавних мікро-взаємодій (збільшення кнопок при наведенні, плавна поява нових транзакцій у списку за допомогою `@keyframes`), що робить користувацький досвід живим.

### 1.4. Програмна логіка: Vanilla JavaScript (ES6+)
Вся логіка додатку написана на чистому JavaScript з використанням найсучасніших можливостей стандарту ECMAScript (ES6+), таких як стрілочні функції, деструктуризація об'єктів, шаблонні рядки (template literals) та класи.
Ключовою архітектурною особливістю проєкту є використання **Native ES Modules** (імпорти та експорти `import/export`). Донедавна для розбиття JS-коду на окремі файли обов'язково вимагалися інструменти на кшталт Webpack. Завдяки підтримці браузерами атрибуту `<script type="module">`, додаток має чисту модульну структуру (`router.js`, `store.js`, компоненти в папці `src/components/`, сторінки в `src/pages/`) без жодного етапу збірки.
Серед інших важливих JS-рішень:
- **Власний Router:** реалізовано механізм перехоплення кліків по посиланнях (`<a>`). Замість переходу браузером, викликається History API (`history.pushState`), змінюється URL, а потім JavaScript динамічно підставляє відповідний HTML-код сторінки у контейнер `<main id="app-content">`.
- **State Management (Керування станом):** оскільки SPA вимагає синхронізації даних між різними екранами, створено глобальний об'єкт `Store` на базі паттерну Observer (Спостерігач). Будь-яке додавання або видалення транзакції через `Store.addTransaction()` автоматично оновлює дані та сповіщає систему про необхідність перемалювання інтерфейсу (наприклад, оновлення загального балансу).

### 1.5. Механізм збереження даних: Web Storage API
Оскільки "Скарбничка" спроєктована як fully-client-side додаток і не вимагає підключення до сервера бази даних, для постійного збереження інформації (persistence) використовується браузерний `localStorage`. 
Цей механізм дозволяє зберігати пари ключ-значення (до 5 МБ даних) локально на пристрої користувача. При кожному оновленні стану `Store`, масив транзакцій серіалізується у рядок за допомогою `JSON.stringify()` і записується в пам'ять браузера. При завантаженні додатку відбувається зворотній процес — десеріалізація через `JSON.parse()`. Це забезпечує повну автономність роботи: дані не втрачаються після закриття вкладки або перезавантаження комп'ютера, гарантуючи безперервність використання програми.

## Висновки

У результаті виконання курсової (практичної) роботи було успішно спроєктовано та розроблено повноцінний веб-додаток "Скарбничка" для ефективного обліку особистих фінансів. Всі поставлені на початку роботи завдання були виконані в повному обсязі.

Створений додаток відрізняється сучасним, інтуїтивно зрозумілим користувацьким інтерфейсом, побудованим за актуальними принципами веб-дизайну (Glassmorphism), що забезпечує високу ергономічність та зручність використання. Завдяки реалізації архітектури Single Page Application (SPA) із застосуванням нативних ES-модулів та кастомного маршрутизатора, вдалося досягти максимальної швидкодії — додаток миттєво реагує на дії користувача без затримок на завантаження сторінок.

Реалізація механізму керування станом (State Management) гарантує консистентність даних у всьому додатку, а використання Web Storage API (`localStorage`) забезпечує надійне, конфіденційне збереження фінансової історії безпосередньо на клієнтському пристрої без необхідності створення серверної інфраструктури.

Таким чином, розроблений програмний продукт повністю відповідає заявленій меті, технічним та бізнес-вимогам, і може бути використаний на практиці для ведення реального особистого бюджету. Крім того, модульна архітектура створює надійний фундамент для подальшого масштабування та вдосконалення системи.
